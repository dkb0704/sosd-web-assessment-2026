# 🚀 sosd web 后端考核任务代码提交仓

欢迎来到 **sosd web 后端组**！本仓库是各位组员提交考核任务代码的专属地方。为了保证代码条理清晰、合并时不产生冲突，请务必在提交代码前**仔细阅读**以下规范。

---

## 📁 目录结构与命名规范

为避免文件冲突，所有人**禁止**直接在根目录或他人目录下创建文件。

所有代码必须提交到 `submissions/` 目录下：

```text
根目录 /
├── submissions/                      # 📌 代码提交总目录 (所有人都在这里提交)
│   ├── 20260101_张三/                 # ✅ 你的作业目录 (命名规范: 学号_姓名)
│   │   ├── src/                      # 你的源码
│   │   ├── pom.xml / package.json     # 你的依赖管理文件
│   │   └── README.md                 # 💡 组员自己编写的运行说明书
│   └── 20260102_李四/
```

> [!IMPORTANT]
> **绝对路径命名格式**：`submissions/<学号_姓名>/`
> 请**不要**加空格、特殊字符，确保文件夹名字里包含你的学号和姓名！

## 🛠️ 第一次提交？（保姆级 Git Flow 教程）

### 1. 克隆仓库到本地

打开终端（Terminal），找一个你喜欢的盘符位置，运行：

```zsh
git clone <本仓库的 Git 链接>
cd sosd-web-assessment-2026
```

### 2. 创建自己的“特性分支” (Feature Branch)

**切记：永远不要在 `main` 分支上直接写代码！**
基于 `main` 分支，创建一个属于你本次任务的新分支：

```zsh
# 格式：feat/学号_姓名_task
git checkout -b feat/20260101_zhangsan_task
```

### 3. 拷贝你的代码至指定目录

1. 在本地找到 `./submissions/` 目录。
2. 在其中创建一个以 `学号_姓名` 命名的文件夹（例如 `20260101_张三`）。
3. 在该目录下进行开发
   * *提示：你的代码根目录下，需要自己写一个 `README.md`，说明你的接口怎么调、数据库怎么连！*

### 4. 提交代码至本地暂存区

```zsh
git add .
git commit -m "feat: 提交考核代码 - 张三"
```

### 5. 推送分支到云端

```zsh
# 只有第一次推送需要加 -u（设置上游分支）
git push -u origin feat/20260101_zhangsan_task
```

### 6. 发起 Pull Request (PR) 🌟

1. 打开浏览器，进入 GitHub/Gitee 仓库页面。
2. 你会看到一个绿色的提示：`Compare & pull request`。点它！
3. **源分支** 选择你的 `feat/...` 分支，**目标分支** 选择 `main`。
4. 填写本次提交的简要说明，点击 **Create pull request**。
5. 等待学期 Code Review，合并（Merge）进 `main`！

## ⚠️ 避坑指南

1. **禁止修改任何人的文件夹及文件**！
2. **务必配置好 IDE 的 `.gitignore`**。本仓库根目录已经内置了通用 `.gitignore`。
   * **不要** 把 `node_modules/`、`target/`、`.idea/`、`.DS_Store` 提交上来，保证仓库体积健康。
3. **每次写代码前，先切回 `main` 分支 `git pull` 一下**，保持本地 `main` 是最新的
